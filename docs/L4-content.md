# L4. コンテンツデザイン

文書番号: SLT-DS-001 §7

UI の文章を、理解・判断・回復を助けるインターフェースとして設計します。

## 共通原則

1. 結論と利用者の次の行動を先に書く。
2. 一文に一つの目的を持たせ、UI に不要な背景説明を詰め込まない。
3. 利用者を責めず、原因、影響、回復手段を示す。
4. 見た目だけの比喩より、操作と結果が誤解されない語を優先する。
5. AI 生成、課金、権限、破壊的操作では不確実性と影響を隠さない。

## ブランド別 voice

| Theme | Voice | 避けること |
|-------|-------|------------|
| AI-DASH | 論理的、簡潔、根拠を示す | 誇張、根拠のない断定、過剰な専門語 |
| Sound Laboratory | 温かく、創造性を支える | 取引だけを強調する表現、抽象的な支援課金 |
| SLT Corporate | 信頼、明瞭、落ち着き | 内輪語、事業間で異なる約束の混在 |
| Admin | 直接的、操作中心、高密度 | 装飾的な長文、重要条件の省略 |

Sound Laboratory の金銭授受は社内方針に従い、価値物語と対象を明確にします（公開面は要約のみ）。

## Action label

- 動詞 + 対象で書く: 「変更を保存」「プロジェクトを削除」。
- 同じ画面で「OK」「実行」「送信」を混在させない。
- 戻るだけなら link、状態を変えるなら button と語彙を一致させる。
- destructive action は対象と不可逆性が分かる label にする。
- loading 中も元の目的が分かる accessible name を保つ。

## 状態メッセージ

| 状態 | 構成 | 例の型 |
|------|------|--------|
| Success | 完了した対象 + 次の状態 | 「設定を保存しました」 |
| Warning | 起きる可能性 + 回避方法 | 「未保存の変更があります。移動前に保存してください」 |
| Error | できなかったこと + 原因 + 回復 | 「保存できませんでした。接続を確認して再試行してください」 |
| Empty | ない理由 + 次の action | 「プロジェクトはまだありません。最初のプロジェクトを作成できます」 |
| Permission | 必要な権限 + 依頼先 | 「閲覧権限が必要です。管理者へ依頼してください」 |

error code や技術詳細は利用者向け本文と分け、support や log で参照できる形にします。

## フォーム文言

- label は入力する情報の名前。
- hint は入力前に必要な形式、範囲、用途。
- placeholder は例示に限定し、label を代替しない。
- required / optional の表記方法をフォーム内で統一する。
- validation は何が不正かと、どう直すかを示す。

## AI 出力

- 「AI による提案」「下書き」など未確定であることを近接表示する。
- 出典がある場合、利用者がたどれる label を付ける。
- confidence を断定表現の代わりに使わず、意味を説明する。
- 再生成で利用者の編集内容が失われる場合は事前に知らせる。
- 外部へ送信・公開する action は人間の確認を言葉でも明示する。

## Inclusive content

- 性別、年齢、能力、文化を不要に仮定しない。
- 色、位置、形だけで対象を指示しない。
- 絵文字や記号だけを accessible name にしない。
- 読み上げ順で意味が通る文にする。
- 略語は初出で展開し、専門用語は [用語集](./L4-terminology.md) と揃える。

## 日付・数値・多言語

- locale formatter を使い、文字列連結で日付・通貨・複数形を作らない。
- timezone を省略すると誤解される予定には zone を表示する。
- 翻訳で長くなることを前提に、button や table header の折返しを確認する。
- 固有名詞、翻訳しない語、表記ゆれは [L4 用語](./L4-terminology.md) に記録する。
- **正本は `ja`。** Docs / Storybook / コンポーネント既定文言の翻訳責務は [ADR-005](./decisions/ADR-005-i18n-locale-architecture.md) に従う（製品 UI は product、横断用語は L4）。
- コード識別子（export・token key・props）は翻訳しない。利用者向けラベルだけを locale の対象にする。
