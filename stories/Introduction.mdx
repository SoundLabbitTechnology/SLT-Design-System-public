# SLT Design System — Storybook

Storybook は、公開 React コンポーネントの API、状態、操作、ブランド差、アクセシビリティを確認するライブカタログです。導入手順、原則、パターン、運用の正本は [Docs index](https://github.com/SoundLabbitTechnology/SLT-Design-System-public/blob/main/docs/README.md) を参照してください。

## 最初の 3 ページ

| Page | 確認すること |
|------|--------------|
| [Theming](./?path=/docs/slt-design-system-l1-foundations-theming--docs) | CSS import、Theme / Mode の契約 |
| [Accessibility](./?path=/docs/slt-design-system-l1-foundations-accessibility--docs) | 自動テストと手動確認 |
| [Landing Canvas](./?path=/story/slt-design-system-samples-landing-canvas--ai-dash-marketing) | 複数 component を組み合わせたブランド判断 |

## コンポーネントカタログ

| Tier | Components |
|------|------------|
| P0 | Button, Input, Textarea, Card, Dialog, Badge |
| P1 | BrandBackground, WaveBackground, GridBackground, Toast, SiteHeader |
| P2 | Skeleton, Table / DataTable |

Tier は優先度です。公開されている全 component は利用可能で、同じ品質基準を満たします。

## Story の見方

1. 左ナビで component を選ぶ。
2. `Docs` で用途、props、全 story を確認する。
3. `Canvas` と Controls で一つの状態を操作する。
4. Toolbar の Theme / Mode を切り替える。
5. Accessibility panel と keyboard で behavior を確認する。

Component の利用判断と Do/Don't は [L2 guides](https://github.com/SoundLabbitTechnology/SLT-Design-System-public/blob/main/docs/L2-components/guides.md) が正本です。

## 変更する場合

- semantic / component token のみを使う。
- public component には主要状態と必要な `play` test を追加する。
- `danger` action は cancel と確認を含める。
- 変更後は `npm run check` と、公開面を変えた場合は `npm run ci` を通す。

手順は [CONTRIBUTING](https://github.com/SoundLabbitTechnology/SLT-Design-System-public/blob/main/docs/CONTRIBUTING.md)、命名と DoD は [Storybook 標準](https://github.com/SoundLabbitTechnology/SLT-Design-System-public/blob/main/docs/L2-components/storybook.md) を参照してください。
